A Guide on How to Analyse Conflict Tasks

Chapter 1 Import Data

Beginners in programming or data analysis often first struggle with importing data. Given that your data is stored locally, you need to ensure that R is pointed to the correct working directory. Otherwise, it won’t be able to find your file. Unlike Python, where the working directory is often implicitly set to the script’s location (depending on the environment), R does not automatically set the working directory to the location of the R script you’re running. Therefore, you must manually set the working directory each time you reopen R or start a new session if you want to run the script successfully.

1.1 Set directory to file path

You can check the current working directory in R using the command getwd(), which stands for get working directory. This will return the current file path—for example, “U:/Documents”. To change the working directory, use the function setwd(), passing the desired file path as a character string. For example: setwd("U:/Documents/Project/Folder/"). To verify that you’ve set the correct directory, you can use the command dir(), which lists all files and folders in the current working directory. Although R Studio provides a graphical interface in the lower-right “Files” panel for navigating and setting directories, it’s generally more reliable and reproducible to explicitly set the working directory using code in your script.

1.2 Import files

Reading data into your R environment can be just as confusing. While R is compatible with many common tabular data formats, such as .csv, .txt, .tsv, .xlsx, and even SPSS (.sav) or Stata (.dta) files, each format requires a slightly different approach and often different functions or packages.

See a list of examples below.

## Examples on how to import different file formats

# Text file
dat <- read.table("textfile.txt")

# csv file
dat <- read.csv("data.csv")
dat <- read.csv2("data.csv")

# Rdata (R-dataframes)
load(file = "file.Rdata")

# SPSS-data
install.packages("foreign") # install required package first
dat <- as.data.frame(foreign::read.spss("SPSSdata.sav"))

# Excel file
install.packages("readxl")
dat <- readxl::read_xlsx("data.xlsx")

# SAS file
install.packages("haven")
dat <- haven::read_sas("SASfile.sas7bdat")

1.2.1 Import csv files

Since experimental data is most commonly stored in .txt or .csv format, we’ll focus on these two. Both formats can be imported using the read.table() function. However, for the data to be read correctly, you typically need to specify two key arguments: sep = ... (which defines the column separator) and header = ... (which indicates whether the first row contains column names). Therefor, I advice to first open the file in a text editor, such as vim, notepad (Mac), or Editor (Windows). It should look something like this:

Figure 1.1: Example csv-file

By opening your data file in a plain text editor, you can inspect how the values are separated (this character is called the delimiter). In the example above, the values are separated by semicolons (;), which means you need to specify sep = ";" when importing the data.

Other common delimiters include:

  • Comma: sep = “,” (typical for .csv files)
  • Tab character: sep = “ (typical for .tsv or some .txt files)
  • Whitespace: sep = “” (for space-separated files)

Additionally, you should set header = TRUE, because the first line of the example shows variable names (ExperimentName;Subject;…). This is usually the case with experimental data.

So, to import the example file, your command would look like this:

dat <- read.table("myfile.csv", header = TRUE, sep = ";")

If you prefer a shortcut, you can use read.csv() or read.csv2(), which are convenient wrappers around read.table() with default values:

  • read.csv() assumes a comma delimiter and header = TRUE
  • read.csv2() assumes a semicolon delimiter and header = TRUE

To check whether your data was imported correctly, you can either run View() or simply click on the dataframe in the Environment tab (top-right panel in R Studio). If the import didn’t work as expected (like the dataframe comprising only one large row), this usually indicates incorrect settings for the header = ... or sep = ... arguments. There could also be (rare) file issues, such as additional text or blank lines, additional delimiters, etc.

Again, either read.csv("yourfile.csv") or read.csv2("yourfile.csv") will usually handle the import correctly.
