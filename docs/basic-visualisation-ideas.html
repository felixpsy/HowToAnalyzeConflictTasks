A Beginners Guide on How to Analyse Conflict Tasks

Chapter 5 Basic Visualisation Ideas

One of the core strengths of R is its ability to produce highly customizable data visualizations. Since plots are a central component of your thesis or manuscript, investing time into creating them is well worth the effort.

While your plots should ultimately reflect and highlight your key findings, it might be best to create them after you have finalized your results. That said, experimenting with different types of plots early on can help you explore the descriptive statistics and distributional patterns in your data, providing insight before any interpretation.

This chapter presents a range of ideas for visualizing data from conflict tasks. If any of the suggestions resonate with you, it’s best to adopt the general concept (or R function) and then build your own plot step-by-step from scratch (just as you learned it - or should learn it anyways)). Simply copying and pasting the code may not work seamlessly, so better try to understand the underlying structure.

If you are looking to improve your data visualization skills, there are plenty of helpful resources online. A good starting point is ggplot2: elegant graphics for data analysis by Hadley Wickham and colleagues, or the general Cookbook on R graphics by Winston Chang.

When visualizing effects in conflict tasks, you have several options. You could plot the mean of different experimental conditions (e.g., interaction plots or barplots), RT distributions (density or violin plots), combine the both approaches (e.g., boxplots or rainbow plots) or plot difference scores [e.g., the CE as RT(incongruent) - RT(congruent)]. But how to choose? Your plot should clearly convey the main message of your thesis—even if that message includes null effects—while balancing between clarity, visual appeal, and informative detail.

5.1 Interaction Plots

Interaction plots are the gold standard for visualizing many effects. Below, you’ll find several options tailored to the example of congruency sequence effects.

Importantly, you should begin by aggregating your data, as your goal is to display four data points—each representing the mean of a specific condition. Crucially, this aggregation must be done at the subject level first (i.e., four means per subject). This step is essential because the number of trials per condition often varies across subjects due to factors like trial exclusions or differing error rates.

5.1.1 Base R Option

In base R, the simplest way to create an interaction plot is by using the built-ininteraction.plot() function (see the R documentation for more details).

Start with your dataframe of aggregated values. Make sure your independent variables (e.g., current trial type and previous trial type) are coded as factors, define your point symbols and y-axis, add axis titles. With just a few lines of code, you’ll have a basic interaction plot.

If you want to add standard errors, you’ll need to calculate them beforehand using the formula sd/sqrt(n) and then include them using the arrows() function.

# Aggregate to 4 data points
dat1 <- aggregate(RT ~ Congruency + Congruency_N_1, datRT, mean)

# Check descriptive data
str(dat1)
## 'data.frame':    4 obs. of  3 variables:
##  $ Congruency    : Factor w/ 2 levels "congruent","incongruent": 1 2 1 2
##  $ Congruency_N_1: Factor w/ 2 levels "congruent","incongruent": 1 1 2 2
##  $ RT            : num  407 471 441 429
# interaction plot
with(dat1, interaction.plot(Congruency_N_1, Congruency, RT, 
                            type="b",
                            pch=c(16, 17), 
                            xlab="Previous Congruency",
                            ylab="RT (in ms)", 
                            ylim = c(400, 500)))


# interaction plot + SE
dat1 <- dat1[order(dat1$Congruency), ]

dat1$sd <- aggregate(RT ~ Congruency + Congruency_N_1, datRT, sd)[, 3]
dat1$n <- aggregate(RT ~ Congruency + Congruency_N_1, datRT, length)[, 3]
dat1$se <- with(dat1, sd/sqrt(n))

arrows(1:2, dat1$RT + dat1$se, 1:2, dat1$RT - dat1$se,
       angle = 90, code = 3, length = .05)

5.1.2 ggplot2 Option

The tidyverse version (using the packages dplyr and ggplot2) of a simple interaction plot is just as straightforward.

# Aggregate  first and calculate Mean, SD, and SE
dat1 <- datRT %>%
  group_by(Congruency, Congruency_N_1) %>% 
  summarize(N = n(),
            mean_RT = mean(as.numeric(RT), na.rm = T),
            sd_RT = sd(RT, na.rm = T),
            se_RT = sd_RT/sqrt(N), .groups = "drop")


# Build the simple interaction plot
ggplot(dat1, aes(x = Congruency_N_1, y = mean_RT, color = Congruency, group = Congruency)) +
  geom_point() +
  geom_line() +
  geom_errorbar(aes(ymin = mean_RT - se_RT, ymax = mean_RT + se_RT), width = 0.1) +
  labs(
    x = "Previous Congruency",
    y = "RT (in ms)",
    title = "Interaction Plot of RT"
  ) +
  theme_minimal()

5.1.2.1 Improve the basic interaction plot

The simple interaction plot can easily be polished into a APA-conform format (code idea is adapted from this osf repository):

# usage of the papaja APA theme for ggplot objects
ggplot(dat1, aes(x = Congruency_N_1, 
                 y = mean_RT, 
                 linetype = Congruency, 
                 group = Congruency)) +
  geom_point(size = 2) +
  geom_line(size = 1) +
  geom_errorbar(aes(ymin = mean_RT - se_RT, ymax = mean_RT + se_RT), 
                width = 0.2, size = 0.5, linetype = "solid") +
  scale_linetype_manual(values = c("solid", "dashed"),
                        labels = c("congruent", "incongruent")) +
  labs(
    x = "Previous Congruency",
    y = "RT (in ms)",
    title = "Interaction Plot of CSE",
    linetype = "Congruency"
  ) +
  papaja::theme_apa() +
  theme(legend.key.width = unit(2, "cm"))

Furthermore, the ggplot2 package offers a variety of handy additional functions, for example, position_dodge can help to prevent overlap between error bars (or points):

# changing the APA Plot, so that error bars do not overlap
ggplot(dat1, aes(x = Congruency_N_1, 
                 y = mean_RT, 
                 linetype = Congruency)) +
  geom_point(aes(group = Congruency), size = 2, 
             position = position_dodge(width = 0.2)) +
  geom_line(aes(group = Congruency), size = 1, 
            position = position_dodge(width = 0.2)) +
  geom_errorbar(aes(ymin = mean_RT - se_RT, 
                    ymax = mean_RT + se_RT, 
                    group = Congruency), 
                width = 0.15, size = 0.5, linetype = "solid",
                position = position_dodge(width = 0.2)) +
  scale_linetype_manual(values = c("solid", "dashed"),
                        labels = c("congruent", "incongruent")) +
  labs(
    x = "Previous Congruency",
    y = "RT (in ms)",
    title = "Interaction Plot of CSE",
    linetype = "Congruency"
  ) +
  papaja::theme_apa() +
  theme(legend.key.width = unit(2, "cm"))

Another great add-on in ggplot2 is facet_grid() — particularly useful when you want to visualize interaction plots across multiple groups (e.g., the CSE for flanker vs. Simon of for experimental vs. control group). facet_grid() allows you to split the plots by one or more additional factors, all in a single, clean layout. The following example shows a flanker vs. Simon task in single task vs. mixed task blocks.

# Aggregate  first and calculate Mean, SD, and SE
dat1 <- datRT %>%
  mutate(
    Congruency = recode(Congruency, 
                        "congruent" = "con", "incongruent" = "inc"),
    Congruency_N_1 = recode(Congruency_N_1, 
                            "congruent" = "con", "incongruent" = "inc")
    ) %>%
  group_by(Congruency, Congruency_N_1, Task_Type, Block_Type) %>% 
  summarize(N = n(),
            mean_RT = mean(as.numeric(RT), na.rm = T),
            sd_RT = sd(RT, na.rm = T),
            se_RT = sd_RT/sqrt(N), .groups = "drop")


# Build the simple interaction plot
ggplot(dat1, aes(x = Congruency_N_1, y = mean_RT, color = Congruency, group = Congruency)) +
  geom_point() +
  geom_line() +
  geom_errorbar(aes(ymin = mean_RT - se_RT, ymax = mean_RT + se_RT), width = 0.1) +
  labs(
    x = "Previous Congruency",
    y = "RT (in ms)",
    title = "Interaction Plot of CSEs across conditions"
  ) +
  theme_classic() +
  facet_grid( ~ Task_Type + Block_Type)

Other additional functions let you customize appearance (e.g., axes, ticks, legend, spacing), add text, combine plots (consider the packages patchwork or cowplot), and much more… Enjoy browsing the many options to be found online :)

5.2 Barplots

Similar to interaction plots, barplots are very commonly found in literature. As always, there are many ways to create barplots. One particular recommendation is using the function bargraph.CI from the sciplot package, as its arguments are straightforward, and it automatically computes and displays within-subject standard errors, saving you a few extra steps. The following example shows a barplot for the proportion congruency effect (PCE) in error rates.

# load the package
library(sciplot)

## define Error Rate from Accuracy
datACC$Err <- 1 - datACC$ACC

## create barplot
bargraph.CI(Task_Prob, Err, group = Congruency, data = datACC[datACC$Task_Type == "Flanker", ],
            legend=TRUE, x.leg = 4, y.leg = 0.25, cex.leg = 1,
            ylab="Error Rate", xlab = "Proportion Congruency",
            main = "PCE Flanker", col = c("lightblue", "lightgrey"),
            ylim = c(0, 0.25), axes = FALSE)
box(bty = "L")
axis(2)

5.3 Density Plots

Although interaction plots (and barplots) are commonly used due to their clear visualization of effects, they are based on aggregated means, which can sometimes obscure important characteristics of the data, such as skewness, outliers, or distributional overlap between conditions. As an alternative, density plots offer a more detailed view of the underlying distribution.

The following example shows how to create a density plot of RTs split by accuracy and congruency. The code is adjusted from a site by Michael Franke.

d_sum <- datACC %>% 
  group_by(ACC, Congruency) %>% 
  summarize(mean_RT = mean(RT),
            sd_RT = sd(RT))
d_sum
## # A tibble: 4 × 4
## # Groups:   ACC [2]
##     ACC Congruency  mean_RT sd_RT
##   <int> <fct>         <dbl> <dbl>
## 1     0 congruent      457.  97.1
## 2     0 incongruent    416. 101. 
## 3     1 congruent      412.  84.8
## 4     1 incongruent    459.  90.7
datACC %>% ggplot(aes(x = RT)) +
  geom_jitter(aes(y = 0.0005), alpha = 0.1, height = 0.0005) +
  geom_density(fill = "gray", alpha = 0.5) +
  geom_vline(data = d_sum, 
             mapping = aes(xintercept = mean_RT), 
             color = "firebrick", size = 0.8) +
  facet_grid(
    Congruency ~ ACC,
    labeller = labeller(ACC = c("0" = "error", "1" = "correct"))
  ) +
  theme_minimal(base_size = 12) +
  theme(
    axis.title = element_text(size = 14, face = "bold"),
    strip.text = element_text(size = 12, face = "bold")
  )

We can also overlap the density functions to display differences in the RT distribution within one plot:

ggplot(datRT, aes(RT, group = Congruency, fill = Congruency)) + theme_bw() +
  geom_density(colour = "black", size = 0.75, alpha = 0.5) +

        axis.text = element_text(size = 10, colour = "black"),
        plot.title = element_text(face = "bold", size = 16),
        legend.title = element_text(size = 10, face = "bold", colour = "black"),
        legend.text = element_text(size = 10, face = "bold", colour = "black")) +
  coord_cartesian(xlim=c(160, 900)) +
  scale_x_continuous(breaks=seq(160,800,100)) +

Or finally, it’s always fun to explore additional, more fancy plotting packages. For instance, we can use the ggridges package to create a ridgeline (or overlapping density) plot. The following example (inspired by Cedric Scherer) shows how to plot reaction time distributions for all trial sequences of the Congruency Sequence Effect (CSE):

library(ggridges)

# define trial sequences first
datRT$Sequence <- paste0(
  ifelse(datRT$Congruency_N_1 == "congruent", "C", "I"),
  ifelse(datRT$Congruency == "congruent", "C", "I")
)

# show four distinct plots
ggplot(
  datRT,
  aes(x = RT, y = Sequence, fill = Sequence)
) +
  geom_density_ridges(
    alpha = 0.8,
    color = "white",
    scale = 2.5,
    rel_min_height = 0.01
  ) +
  coord_cartesian(xlim = c(160, 900)) +
  scale_x_continuous(breaks = seq(160, 800, 100)) +
  scale_fill_manual(
    values = c("CC" = "#A6CEE3", "CI" = "#1F78B4", "IC" = "#2171B5", "II" = "#08306B")
  ) +
  labs(
    title = "Densities underlying the CSE",
    y = "Trial Sequence",
    x = "Response latencies in ms"
  ) +
  guides(fill = "none") +
  theme_ridges() +
  theme(
    axis.title = element_text(size = 10, face = "bold", colour = "black"),
    axis.text = element_text(size = 10, colour = "black"),
    plot.title = element_text(face = "bold", size = 16)
  )

5.4 Combined Plots

Ultimately, it might be helpful to combine showing the means of your experimental conditions with visualizations of the full distribution of your dependent variable. For instance, instead of a simple barplot, consider using a violin plot paired with a boxplot (see the first example of Allen et al., 2021, the authors also provide code).

Taking this a step further, inspired by a figure from Schiltenwolf et al., 2022 (to be found in Schiltenwolf, 2024, p. 93) ), the following example combines an interaction plot with both violin and bar plots in a single visualization:

# create a dataset first to rename factors and to filter Block Type
datRT_plot <- datRT %>%
  mutate(
    Congruency = recode(Congruency,
                        "congruent" = "con", "incongruent" = "inc"),
    Congruency_N_1 = recode(Congruency_N_1,
                            "congruent" = "N-1 con", "incongruent" = "N-1 inc"),
    Congruency = factor(Congruency, levels = c("con", "inc")),
    Congruency_N_1 = factor(Congruency_N_1, levels = c("N-1 con", "N-1 inc")),
    Task_Type = factor(Task_Type))  %>%
  filter(Block_Type == "single")

# create the plot
ggplot(datRT_plot, aes(x = Congruency_N_1, y = as.numeric(RT), fill = Congruency)) +
  geom_violin(position = "identity", alpha = 0.4, width = 0.9, color = NA) +
  geom_boxplot(position = position_dodge(0.4), width = 0.15, outlier.shape = NA) +
  stat_summary(fun = mean, geom = "point", aes(group = Congruency),
               position = position_dodge(width = 0.4), shape = 21, 
               fill = "black", size = 2) +
  stat_summary(fun = mean, geom = "line", aes(group = Congruency),
               position = position_dodge(width = 0.4), color = "black", 
               size = 0.5) +
  facet_grid( ~ Task_Type) +
  labs(x = "", y = "Reaction Times (in ms)") +
  theme_minimal(base_size = 14) + 
  theme(
    panel.border = element_rect(color = "black", fill = NA, linewidth = 1),
    strip.background = element_rect(fill = "#2c3e50"),
    strip.text = element_text(color = "white", face = "bold"),
    legend.position = "bottom",
    legend.box = "horizontal",
    legend.text = element_text(size = 10),
    legend.key.height = unit(0.3, "cm"),
    legend.margin = margin(t = -5),
    legend.box.margin = margin(t = 5)) +
  scale_fill_manual(values = c("#3498db", "#2ecc71"))

While this plot conveys lots of information about the data, its main message may not be immediately clear. So in the next section, we’ll explore how to visualize difference scores to present effects most straightforwardly (as also done by Schiltenwolf and colleagues in a second plot).

5.5 Plotting mean differences

A plot of mean differences is often the clearest way to highlight the effect you’re interested in. Keep two points in mind: first, this type of visualization can be more complex to code, since you may need to build it from scratch rather than relying on pre-built functions. Second, it’s recommended to complement a mean differences plot with another visualization (such as an interaction plot, density plot, or a combined plot), to show what contributes to those differences.

The following example illustrates a comparison of the post-error slowing effect between single-task blocks (flanker or Simon) and mixed-task blocks (both flanker and Simon). The general approach is that you calculate the effect first (this could also apply to effects like the CSE, switch costs, PCE, etc.) and then plot it as a barplot with SEs.

# set the number of participants
N <- 4

# aggregate data, so you can calculate RT(N-1 Error) - RT(N-1 Correct) per subject
dat1 <- aggregate(RT ~ Subject + ACC_N_1,
                  datRT[datRT$Block_Type == "single",], mean)

# create an empty dataframe to store the individual values of post-error slowing
dat_PES_single <- data.frame(
  Subject = unique(dat1$Subject),
  RT = NA
)

# loop through all subjects and calculate PES for each one
for (i in (unique(dat1$Subject))) {
  dat3 <- dat1[dat1$Subject == i, ]
  dat_PES_single$RT[dat_PES_single$Subject == i] <- diff(dat3$RT) * -1
}

# store mean and SE of PES as objects
mean_single <- mean(dat_PES_single$RT)
se_single <- sd(dat_PES_single$RT) / sqrt(N)


## repeat all steps for the mixed blocks
dat2 <- aggregate(RT ~ Subject + ACC_N_1,
                  datRT[datRT$Block_Type == "mixed",], mean)

dat_PES_mixed <- data.frame(
  Subject = unique(dat2$Subject),
  RT = NA
)

for (i in (unique(dat2$Subject))) {
  dat3 <- dat2[dat2$Subject == i, ]
  dat_PES_mixed$RT[dat_PES_mixed$Subject == i] <- diff(dat3$RT) * -1
}

mean_mixed <- mean(dat_PES_mixed$RT)
se_mixed <- sd(dat_PES_mixed$RT) / sqrt(N)


# combine both scores in one barplot
barplot(height = c(mean_single, mean_mixed),
        names.arg = c("Single task blocks", "Mixed task blocks"),
        ylim = c(0, 80),
        xlab = "", ylab = "Post-Error Slowing (ms)",
        space = 0.5, col = c("lightblue", "#F08080"))
# add SE bars
arrows(x0 = c(1, 2.5), y0 = c(mean_single - se_single, mean_mixed - se_mixed),
       y1 = c(mean_single + se_single, mean_mixed + se_mixed),
       angle = 90, code = 3, length = .3, lwd = 2)

abline(h = 0.0)

By combining this approach with the a plot from the previous section, we can create an visualization that balances transparency and clarity: Plot A shows the underlying data structure, while Plot B clearly highlights the effect of interest. This combination comes close to the ideal of a well-rounded, informative plot.

5.6 Final notes on graphics

When working with graphics in R, it’s a good practice to save your plots as objects rather than copying them directly from RStudio’s plot pane. This allows you to export high-resolution images using functions like ggsave() for ggplot2 objects or png() / pdf() for base R plots.

If you’re creating multiple related plots (e.g., error rates and reaction times), maintain consistency in colors and styles across all figures (e.g., see the last example).

If you want to play around with colors, I recommend checking out the R Graph Gallery’s Color Palette Finder. Once you find a palette you like, just click the download icon to export the hexadecimal color codes for easy use in your plots. Another resource is ColorBrewer, which offers a variety of well-designed color schemes. Just keep in mind that colors may look different when printed on paper.
